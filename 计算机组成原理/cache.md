# Cache



## 软件实现的cache



### redis的cache

对于java后端，为了减少数据库的压力，常常需要配备一个cache层，用户对于数据的访问，是直接访问cache，如果找到了，那么直接返回结果，如果没有找到，那么就去数据库中拿到数据，返回给用户，并且将其存在cache中。

redis实现的cache，是以键值对的形式加载在内存中，因为是纯内存操作，Redis的性能非常出色，每秒可以处理超过 10万次读写操作
